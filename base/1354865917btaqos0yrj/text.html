В следующем списке приводятся минимальные требования к оборудованию и программному обеспечению для запуска Microsoft SQL Server 2005.

Примечание.

Требования для запуска SQL Server на 32-разрядной платформе отличаются от требований к 64-разрядной платформе.

Требования к оборудованию и программному обеспечению (для 32-разрядной и 64-разрядной платформ)

Монитор

Для работы графических средств SQL Server требуется разрешение VGA или выше. Минимальное разрешение: 1024x768 точек.

Указывающее устройство

Требуется мышь Microsoft mouse или совместимое указывающее устройство.

Дисковод для компакт-дисков или DVD-дисков

Для установки с компакт-дисков или DVD-дисков требуется соответствующий дисковод.

Требования к кластерному оборудованию

На 32-разрядных и 64-разрядных платформах поддерживается установка кластера из восьми узлов (то есть максимального количества узлов, поддерживаемого Microsoft Windows Server 2003). Дополнительные сведения о решениях c высоким уровнем доступности см. в разделах Настройка высокого уровня доступности и Подготовка к установке отказоустойчивого кластера.

Требования к сетевому программному обеспечению

Требования к сетевому программному обеспечению для 64-разрядных версий SQL Server 2005 те же, что и для 32-разрядных версий. Windows Server 2003, Windows XP и Windows 2000 имеют встроенное сетевое программное обеспечение.

Примечание.

SQL Server 2005 не поддерживает протокол Banyan VINES Sequenced Packet (SPP), а также сетевые протоколы Multiprotocol, AppleTalk и NWLink IPX/SPX. Клиенты, подключенные ранее с помощью этих протоколов, для соединения с SQL Server 2005 должны выбрать другой протокол.

Изолированные именованные экземпляры и экземпляры по умолчанию поддерживают следующие сетевые протоколы:

  • Общая память
  • Именованные каналы
  • TCP/IP
  • VIA

Примечание.   Общая память не поддерживается отказоустойчивыми кластерами.

Требования к Интернет-компонентам для 32- и 64-разрядных версий SQL Server 2005 одинаковые. В следующей таблице приводятся требования к Интернет-компонентам для SQL Server 2005.

Программное обеспечение для Интернета1

Для всех выпусков SQL Server 2005 требуется Microsoft Internet Explorer 6.0 с пакетом обновления 1 (SP1) или более поздняя версия, так же как для консоли управления MMC и справки HTML. Достаточно минимальной установки Internet Explorer. Кроме того, Internet Explorer не обязательно должен быть обозревателем по умолчанию.

Но если устанавливаются только клиентские компоненты и устанавливаются соединения с сервером, не требующие шифрования, достаточно будет Internet Explorer 4.01 с пакетом обновления 2 (SP2).

Службы IIS

Для установки служб Microsoft SQL Server 2005 Reporting Services (SSRS) требуется IIS 5.0 или более поздняя версия.

Дополнительные сведения об установке IIS см. в разделе Как установить службы Microsoft IIS. Дополнительные сведения об установке IIS 7.0 на Windows Vista см. на веб-узле Майкрософт.

Дополнительные сведения о настройке IIS см. в разделе Как установить 32-разрядную версию служб Reporting Services на 64-разрядный компьютер.

Требования к программному обеспечению

Для установки SQL Server требуется установщик Microsoft Windows 3.1 или более поздняя версия, а также компоненты доступа к данным MDAC 2.8 с пакетом обновления 1 (SP1) или более поздней версии. Компоненты MDAC 2.8 с пакетом обновления 1 (SP1) можно загрузить с этого веб-узла Майкрософт.

Программа установки SQL Server устанавливает следующие необходимые компоненты:

  • Microsoft .NET Framework 2.0
  • Собственный клиент Microsoft SQL Server
  • Файлы поддержки программы установки Microsoft SQL Server.

Примечание.

SQL Server 2005 Express Edition не устанавливает .NET Framework 2.0.Перед установкой SQL Server 2005 Express Edition необходимо загрузить и установить платформу .NET Framework с этого веб-узла Майкрософт.

Если она еще не установлена, программа установки SQL Server устанавливает каждый из этих компонентов отдельно. При отмене установки SQL Server 2005 автоматически удаляются только файлы поддержки программы установки Microsoft SQL Server. Дополнительные сведения об удалении этой версии см. в разделе Как удалить существующий экземпляр SQL Server 2005 (программа установки).

Примечание.

Пакет средств разработки .NET Framework 2.0 (SDK) не устанавливается при установке SQL Server 2005. Пакет SDK содержит документацию, компилятор C++ и другие инструменты, полезные при использовании .NET Framework для разработки в SQL Server. Пакет .NET Framework SDK можно загрузить с этого веб-узла Майкрософт.

После установки нужных компонентов SQL Server программа установки проверяет соответствие компьютера, на котором устанавливается SQL Server, всем остальным требованиям. Дополнительные сведения см. в разделе Параметры для средства проверки конфигурации системы.

Отказоустойчивым кластерам SQL Server 2005 необходимо наличие Microsoft Cluster Server (MSCS) по крайней мере на одном узле кластера серверов. Сервер MSCS поддерживается только в случае, если он был установлен на компьютер, конфигурация которого была протестирована на совместимость с программным обеспечением MSCS. Дополнительные сведения см. в разделе Подготовка к установке отказоустойчивого кластера.

Поддержка версий на разных языках

Обновление:

  • Английские версии SQL Server можно обновить до любой локализованной версии SQL Server 2005.
  • Локализованные версии SQL Server можно обновить до локализованных версий SQL Server 2005 на том же языке.
  • Локализованные версии SQL Server нельзя обновить до английской версии SQL Server 2005.
  • Локализованные версии SQL Server нельзя обновить до локализованных версий SQL Server 2005 на другом языке.

Все локализованные обновления должны соответствовать поддерживаемому порядку обновления. Дополнительные сведения о поддерживаемом порядке обновления см. в разделе Обновления версий и выпусков.

Дополнительная поддержка версий на разных языках:

  • Английская версия SQL Server 2005 поддерживается на всех локализованных версиях поддерживаемых операционных систем.
  • Локализованные версии SQL Server 2005 поддерживаются на локализованных операционных системах на том же языке, что и локализованная версия SQL Server.
  • Локализованные версии SQL Server 2005 также поддерживаются в английских версиях поддерживаемых операционных систем с помощью пакета многоязыкового пользовательского интерфейса Windows. Однако необходимо проверить определенные настройки операционной системы перед установкой локализованной версии SQL Server 2005 на сервер под управлением английской версии операционной системы с настройками MUI, отличными от английского языка. Необходимо убедиться, что следующие настройки операционной системы соответствуют языку локализации: SQL Server
  • Настройки пользовательского интерфейса операционной системы
  • пользовательские настройки языкового стандарта операционной системы;
  • настройки языка системы.

Если эти параметры операционной системы не соответствуют языку локализованной версии SQL Server, необходимо правильно настроить эти параметры операционной системы, как описано в разделе Как изменить параметры операционной системы для поддержки локализованных версий.

Поддержка виртуальной операционной системы

SQL Server 2005 можно установить на виртуальных компьютерах Microsoft Virtual Server и Microsoft Virtual PC на поддерживаемых операционных системах и оборудовании. Сведения о Virtual Server см. на веб-узле Майкрософт. Сведения о продукте Virtual PC см. на веб-узле Майкрософт.

Внимание!

Не устанавливайте SQL Server 2005 на Microsoft Virtual Server или Microsoft Virtual PC, работающих на оборудовании, которое не приводится в следующем списке «Требования к оборудованию и программному обеспечению».

Требования к свободному месту на жестком диске (32-разрядная и 64-разрядная версии)

Во время установки SQL Server 2005 установщик Windows создает временные файлы на системном диске. Прежде чем запускать программу для установки или обновления версии до SQL Server 2005, проверьте, что на нем доступно не менее 2,0 ГБ свободного места. Это требование применяется даже в том случае, если компоненты SQL Server устанавливаются на диск, отличный от предложенного по умолчанию. Многие файлы устанавливаются на системный диск, как правило, диск C:. Например, файлы журналов программы установки будут записаны на системный диск и могут потребовать 80 МБ для изолированной установки и больше для установки отказоустойчивого кластера. Дополнительные сведения см. в разделе Расположение файлов для экземпляра по умолчанию и именованных экземпляров SQL Server 2005.

Фактические требования к свободному месту на жестком диске зависят от конфигурации системы, а также от устанавливаемых приложений и компонентов. В следующей таблице представлены требования к свободному месту на диске для компонентов SQL Server 2005.

При установке на поддерживаемые операционные системы могут возникнуть следующие проблемы и ограничения:

  • встроенная поддержка веб-служб (SOAP и HTTP) доступна только для экземпляров SQL Server 2005 под управлением Windows Server 2003;
  • отдельные разделы в документации по службам Microsoft SQL Server 2005 Integration Services (SSIS) по программированию, объектам AMO и ADOMD.NET могут указывать на поддержку ранних версий Windows, например Windows 98, Windows ME или Windows NT 4.0. Тем не менее, в этой версии эти три программных интерфейса поддерживаются только операционными системами Windows XP, Windows 2000 и Windows Server 2003;
  • Отказоустойчивым кластерам SQL Server 2005 необходимо наличие Microsoft Cluster Server (MSCS), по крайней мере на одном узле кластера серверов. Сервер MSCS поддерживается только в случае, если он был установлен на компьютер, конфигурация которого была протестирована на совместимость с программным обеспечением MSCS. Дополнительные сведения см. в разделе Подготовка к установке отказоустойчивого кластера.

Поддержка расширенных систем

64-разрядные версии SQL Server 2005 включают поддержку расширенных систем, также известных как Windows on Windows (WOW64). WOW64 — это функциональная возможность 64-разрядных выпусков Microsoft, позволяющая 32-разрядным приложениям выполняться в своем 32-разрядном режиме. Приложения работают в 32-разрядном режиме даже несмотря на то, что базовая операционная система выполняется на 64-разрядной платформе.
